# Условные задания — Стандарты

Правило отбора: **в этом файле только то, что можно проверить машиной или тестом.** Всё, что проверяется вкусом, — не стандарт, а мнение, и здесь ему не место.

## Границы (проверяются CI, не ревью)

| Граница | Как заявлена | Как проверена |
|---|---|---|
| `logic` не делает I/O | ноль зависимостей, время — аргумент | `cargo tree -p conditional-tasks-logic` печатает один крейт; `grep -rE 'ic_cdk\|std::(fs\|net\|time)\|reqwest' logic/src/` пусто |
| `logic` не знает про чейны и криптографию | адреса и ключи — непрозрачные байты | `grep -riE 'evm\|solana\|keccak\|secp256k1\|ed25519\|candid' logic/src/` пусто |
| Канистра не двигает деньги | нет переводов, нет знания о сплиттере | `grep -riE 'transfer\|approve\|splitter\|fee\|bps\|treasury' canister/src/` пусто |
| Канистра не читает внешние сети | нет outcalls, нет RPC-канистр | `grep -r 'http_request' canister/src/` пусто; внешние вызовы только к management canister и `crown_index` из конфига |
| Update-поверхность фиксирована | шесть методов | парсер `.did` падает на update вне `{register_task, accept, decline, done, vote, set_channel_params}` |
| Авторизация не по principal | только подпись кошелька | `grep -r 'caller()' canister/src/` не встречается в проверках прав |
| Ядро и фабрика не тронуты | отдельный репозиторий | CI падает, если в PR есть путь `crown-core/` или `crown-factory/` |

Зависимости строго в одну сторону: `canister → logic`, `canister → crown-derive`. Обратных нет. `logic` о существовании канистры не знает.

## Код

**`logic`:**
```rust
#![forbid(unsafe_code)]
#![deny(clippy::unwrap_used, clippy::expect_used, clippy::panic,
        clippy::arithmetic_side_effects, clippy::indexing_slicing)]
```
Только `checked_*` арифметика. Исчерпывающие `match` по `(State, Action)`, никаких `_ =>` на доменных enum. Ошибка — значение, не паника. Время — аргумент функции; `logic` не имеет понятия «сейчас».

**`canister`:** `clippy` без предупреждений. `unwrap`/`expect` запрещены на путях ingress и таймеров; в `#[init]` допустимы — паника там означает несостоявшийся деплой. Вердикт пишется в стабильную память **до** запроса threshold-подписи; порядок закреплён тестом.

**Криптография:** только пиннутые библиотеки (`k256`, `ed25519-dalek` или эквиваленты). Ничего самописного: ни хешей, ни recover, ни разбора подписей. Байтовые форматы digest/message (game-spec §8) собираются в одном модуле и пиннутся юнит-тестами на конкретные векторы.

**Общее.** У каждой строки есть текущий потребитель. Ни мёртвого кода, ни обобщённости «на будущее», ни фича-флагов. Сомневаешься между «добавить» и «удалить» — удаляй.

## Тесты, без которых этап не закрывается

1. Машина состояний: недостижимость, единственность вердикта, большинство, дедуп, время, детерминизм — property-тестами в `logic`.
2. `task_id`: паритет с `crown-derive` на векторах `crown-factory` и фаззом; на G4 — с адресом реально рождённого эскроу.
3. Подписи участников: чужой ключ, чужое задание, чужая канистра, чужое действие, повтор счётчика — отвергнуты.
4. Байты вердикта: локальный recover/verify + принятие реальным контрактом на обеих тестовых сетях (e2e).
5. Один вердикт навсегда: повторный тик, повторный запрос, запрос другого исхода — ничего не меняют.
6. Вес голоса: порог, чужая сеть, повтор, отказ `crown-index` без потери голоса.
7. Сертификат состояния проверяется против root key NNS.
8. Изоляция: `git diff` в `crown-core` и `crown-factory` пуст.

Тесты лежат рядом с исходником, который проверяют.

## Конфиг

Единая `config/`. Ничего сетевого в коде.

Два профиля: `testnet`, `mainnet`. Смена среды = смена профиля, ноль изменений кода.

Ровно ключи из game-spec §7: `crown_index`, `threshold_key`, `voting_period`; per-chain — `id`, `factory`, `escrow_init_code_hash` (EVM), `evm_chain_id` (EVM), `domain` (Solana), `min_gross`. Всё.

Нет и не появится: RPC-URL, API-ключ, адрес сплиттера, `feeBps`, казна, адреса кошельков, `duration`, `deadline` — последние два задаёт донор в каждом задании.

Значения `factory`, `escrow_init_code_hash`, `min_gross` берутся из `crown-factory/deploy/`, `crown_index` — из деплоя ядра. Этот репозиторий их не порождает — только копирует.

## Циклы

Канистра на внешних сетях газ **не платит вообще** и ключей для транзакций не держит.

- **Газ за `createEscrow` — донор. За `claim` — кто нажал** (открытое решение game-spec §12).
- **Циклы канистры — оператор.** Расход: ingress участников, межканистерные вызовы к `crown-index`, threshold-подписи (заметно дороже остального), таймеры, certified query. Бюджет — G5.
- Ingress с подписями — вектор спама, жгущего циклы. Принято (game-spec §11.4); митигация — rate limit на boundary nodes и бюджет, не код.

## Именование

- Файлы кода: kebab-case. Типы: PascalCase. Функции/переменные: snake_case. Константы: UPPER_SNAKE_CASE.
