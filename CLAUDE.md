# Crown Games — Условные задания — CLAUDE.md

Этот репозиторий содержит **только игру «Условные задания»**. Один артефакт:

1. **`conditional-tasks`** — канистра ICP. Ведёт задания, принимает подписи участников, считает голоса весом из книги, подписывает вердикт threshold-ключом и отдаёт его через `query`. Денег не держит, транзакций не шлёт, внешние сети не читает.

Контрактов здесь нет: игра работает на задеплоенной форме `two-outcome` из `crown-factory`. Ядро (`crown-core`) и фабрика (`crown-factory`) не трогаются ни одной строкой. Сервер, фронт, тексты, модерация — `crown-app`, не здесь. См. `docs/post-game.md`.

Этот файл читается каждую сессию. Держи коротким.

## Незыблемые принципы

1. **Игра — это резолвер и только.** Публичный ключ плюс правила, по которым он подписывает. Write-канала «игра → ядро» не существует: единственное влияние на книгу — честный сеттлмент эскроу.
2. **Канистра не двигает деньги.** Ни транзакций наружу, ни ключей внешних сетей помимо threshold-подписи вердикта, ни нативного токена. Вердикт — подпись; исполняет её кто угодно транзакцией `claim`.
3. **Канистра не читает внешние сети.** Ни RPC-канистр, ни HTTPS-outcalls. Всё об эскроу заявлено при регистрации; вердикт для недофинансированного эскроу бесполезен — деньги двигает только эскроу.
4. **Один эскроу — один вердикт, навсегда.** Вердикт пишется в стабильную память **до** запроса подписи и не переписывается. Двух подписанных исходов одного эскроу не существует.
5. **Границы — компилятором, не договорённостью.** `logic` — крейт с нулём зависимостей; время — аргумент функции, не системный вызов. Он физически не может сделать I/O. Проверяется CI, не ревью.
6. **Авторизация — подпись кошелька, не вызывающий.** Ingress шлёт кто угодно; право даёт подпись `(canister_id, task_id, действие)` кошельком той сети, где живёт эскроу. Сервер не стоит ни в одном пути.
7. **Вес голоса — книга, и только книга.** Межканистерный вызов `crown-index`. Порог и вес — накопленная репутация, не платёж. Правило вердикта: строгое большинство веса, иначе `cancel`.
8. **Байты подписи вердикта — интерфейс, а не деталь.** digest EVM и message Solana формы `two-outcome` воспроизводятся побайтово (game-spec §8); расхождение = деньги заперты до `refund`.
9. **Минимализм = безопасность.** У каждой строки есть текущий потребитель. Никакой обобщённости «на будущее». Сомневаешься между «добавить» и «удалить» — удаляй.

Расхождение кода с принципами или с `docs/` = баг. Не «решаем на месте»: правь код; если неверен принцип — сначала правь доку.

## Структура репозитория

```
conditional-tasks/
├─ CLAUDE.md
├─ docs/
├─ logic/                  # КРЕЙТ БЕЗ ЗАВИСИМОСТЕЙ. Машина состояний, голоса, вердикт.
│  └─ src/{lib,task,vote,verdict}.rs
├─ canister/               # КАНИСТРА. Ingress, подписи, вес, threshold, сертификация.
│  ├─ conditional-tasks.did
│  └─ src/{lib,api,auth,certify}.rs   # weight.rs — G3, sign.rs — G4
├─ config/
│  ├─ testnet.toml
│  └─ mainnet.toml
└─ scripts/
```

Зависимости строго в одну сторону: `canister → logic`, `canister → crown-derive` (внешний крейт из `crown-factory`). Обратных нет. `logic` не знает про ICP, про чейны, про криптографию, про Candid.

## Как работать

1. Прочитай `docs/build-plan.md`, определи текущий этап (первый не-`DONE`).
2. Работай строго по нему. Не забегай вперёд.
3. Этап не закрыт, пока Definition of Done не выполнен и тесты зелёные.
4. По завершении — обнови статус, удали устаревшую доку тем же коммитом, только потом следующий этап.

## Гардрейлы (NEVER)

- Никогда не добавляй в `logic` зависимость. Ни одну. Время, адреса, хеши приходят аргументами; конверсия на границе в `canister`.
- Никогда не давай канистре двигать деньги: ни `transfer`, ни `approve`, ни знания о сплиттере, казне, `feeBps`. Единственное число про деньги — флор формы из конфига.
- Никогда не читай внешние сети: ни RPC-канистры, ни `http_request`. Единственные внешние вызовы — management canister (threshold-подпись) и `crown-index` из конфига.
- Никогда не подписывай второй исход для эскроу. Вердикт — в стабильную память до запроса подписи; ретрай допустим только для того же исхода.
- Никогда не подписывай вердикт вне точных байтовых форматов формы `two-outcome` (game-spec §8).
- Никогда не авторизуй действие по вызывающему principal. Только подпись кошелька, привязанная к `(canister_id, task_id, действие)`.
- Никогда не принимай текст задания в канистру — только `sha256(text ‖ text_salt)`. Текст живёт на сервере.
- Никогда не изменяй и не удаляй записанный голос, вердикт или подпись участника.
- Никогда не добавляй update-метод сверх фиксированного списка в `.did`. Новый update = новая доверенная поверхность = сначала правка спеки.
- Никогда не переноси состояние между версиями канистры. Новая канистра стартует пустой; старая доживает свои задания.
- Никогда не хардкодь адреса, chain id, principal-ы. Только `config/`.
- Никогда не трогай `crown-core` и `crown-factory`. Нужна другая форма эскроу — это их репозитории и их правила.
- Никогда не добавляй код «на будущее» без текущего потребителя. Если хочется — это `docs/post-game.md`.

## Стек

- `logic`: Rust, zero deps, `#![forbid(unsafe_code)]`, только `checked_*` арифметика.
- `canister`: Rust, `ic-cdk`, `ic-stable-structures`, `crown-derive`; криптография — только пиннутые библиотеки (`k256`, `ed25519-dalek` или аналоги), ничего самописного.
- Тесты: property-тесты в `logic`, PocketIC для канистры, e2e-скрипты против Sepolia и devnet.
- Доки и проза — по-русски. Идентификаторы, комментарии, имена файлов кода — по-английски.
